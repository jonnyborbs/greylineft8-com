Release Notes

What's new in each version of Greyline FT8.

Version 1.0.10 Latest
Fixes
  • Minor change to a watchdog timer released in 1.0.9 to remove the risk of an extended TX cycle
Version 1.0.9
Shoutouts
  • K4RZOMy favorite coyote — for identifying and helping me chase down some audio pipeline restore and timing bugs
Features
  • Big CTY data now automatically updates. We do a freshness check on each app launch, and force an update approximately once a week. Data loads are only about 300kb so bandwidth requirements are tiny.
  • Added a Time Debugging panel in the Settings menu. Most of you will never need this. But in case you see mis-timed messages, high instances of drift, or missed QSOs this screen will provide me the data I need to help track down the issue and make a better app for everyone.
Fixes
  • The audio pipeline now gracefully resumes when it's interrupted — whether you accidentally activated Siri, unplugged/switched audio devices, or started the latest episode of your favorite Star Trek Podcast by mistake. Greyline now detects and compensates for these interruptions.

Thanks and 73 to everyone for the feedback and bringing me along on your activations!

Version 1.0.8
Shoutouts
  • KN7AHSIdentifying a missing field in our ADI exports
Features
  • Greyline now makes use of Big CTY data courtesy of AD1C and ClubLog (country-files.com)
  • This new data source makes grid square, call sign, State/Province/Country and DXCC lookup fast, reliable, and fully local with no internet access required
  • Greyline now supports both QRZ and HamQTH callsign lookups. If you're an existing user, make sure to check out the Settings page and input your login for one or both!
  • Antenna tune mode now automatically applies suggested free frequencies
Fixes
  • ADIF exports now include the DXCC numeric code for every QSO
  • ADIF exports now include the Country identifier for every QSO, including non-US ones (whoops!)
  • Significant streamlining to log file handling and guard rails for bad or malformed data

Thanks and 73 to everyone for the feedback and bringing me along on your activations!

Version 1.0.7
Shoutouts
  • KJ5FVIRequesting selective log export
  • M0RVB, G7UFOSuggesting "Tune" functionality
Features
  • You can now select only specific QSOs to export to ADI from your log
  • Exporting to file will place a marker in your log showing when you last exported and how many log lines were saved
  • Now that we have this marker, there's also a simple "Export all since last" option that will save off all your logged entries since the last export to a new file — great for uploading to POTA.app or your external logger of choice
  • Added a new "Antenna tune" function — this function will transmit a constant carrier for a predefined amount of time. You can select the frequency from the main control panel, and the transmit duration from either the Settings menu or the Antenna Tune screen directly
  • The Tune screen will show a large-font countdown (or count-up, if you selected an unlimited time) so you can clearly see how long you have left if you walk away from your station to touch an antenna 25 feet away

Thanks and 73 to everyone for the feedback and bringing me along on your activations!

Version 1.0.6
Shoutouts
  • K0JGLReporting grid square lookup issues
  • IW2EYNSuggesting log import functionality
Features
  • Added the ability to import ADI formatted log files. Imported log entries will help you know who you've worked before. Note that imported log entries CAN be exported to a new ADI file, but can NOT be uploaded to QRZ — this is intentional, to prevent conflicts and duplication.
Fixes
  • Grid squares were occasionally returning partially correct lookup results, due to an "alphabetically first wins" algorithm where multiple states, provinces or countries were in a single grid. Now we show everyone — no favoritism here!
  • When you highlight a log row, we now use much more readable and high-contrast colors

Thanks and 73 to everyone for the feedback and bringing me along on your activations!

Version 1.0.5
Fixes
  • When applying a manual override TX frequency, the frequency selection behavior changes to "Hold" and respects your selection. The app will always return to Auto Select mode on startup, since band conditions are always changing.
  • Applied dynamic delay algorithm to FT4 cycles to eliminate dropped symbols with high-latency audio devices. FT4 decoding is now SUPER reliable!
  • Always check previous band activity for grid squares. I never want to see a log entry without one again!

Thanks and 73 to everyone for the feedback and bringing me along on your activations!

Version 1.0.4
Shoutouts
  • K0ARGReporting issues with FT4 timing
Features
  • Added an option to preselect Previous, FT8 or FT4 as your startup mode
  • Added a "Mode" selector to the logging dialog so you can pick FT8 or FT4 when creating a manual entry
Fixes
  • FT4 timing is now corrected and operates on the 7.5s cycle rather than FT8's 15s one
  • When FT4 mode is active, we now correctly display frequency settings at the top of the band activity view
  • When tapping a message to restart a dropped or timed out QSO, R-signal reports are now correctly sent with the R prefix
  • Ensured the 160m and 60m bands are not selectable for FT4 per the band plan

Thanks and 73 to everyone for the feedback and bringing me along on your activations!

Version 1.0.3
Shoutouts
  • KL7ECSupport for TX-500 confirmed, now validating no spaces in callsigns
  • NL7OTSupport for QRP Labs QMX confirmed
  • IW2EYNString validation in QRZ API keys, great club promotion!
  • KO6IUHUpdated connection info for IC-705
Features
  • Added a decrementing counter in the status area, so you can see how many times a message is due to be retransmitted before we time out
Fixes
  • If the message being sent changes in the middle of a TX cycle, update the band activity view with the new message
  • When the retransmission counter is reached while in Gatherer mode, we now return to calling CQ instead of halting transmission
  • When the retransmission counter is reached while in Hunter mode, we now call the next station on the next available TX slot instead of waiting several cycles first
  • Sometimes, grid squares were not logged while in Gatherer mode. Now they should be.
  • String validation in the QRZ API key and Callsign fields, so phantom spaces should no longer cause problems

Thanks and 73 to everyone for the feedback and bringing me along on your activations!
